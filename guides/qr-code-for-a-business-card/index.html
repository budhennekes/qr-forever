How to Make a QR Code for a Business Card

By Bud Hennekes, who built QR Codes Made Easy. Updated July 2026.

The best business-card QR code points at a web page you control (your site or a simple contact page), not a fixed vCard, so you can update your details without reprinting cards. Make it static so it never expires, keep it around 0.6 to 0.8 inches with the quiet margin intact, and use dark on light for a clean close-up scan.

What should it link to? (URL vs vCard)

You have two real options, and the choice shapes everything else.

A URL code points at a page. That could be your site, your portfolio, a simple contact or landing page, or your booking link. The person scans, their browser opens the page, and they do whatever you set up there.

A vCard code stuffs your contact details straight into the pattern. Name, number, email, all of it lives in the squares. The upside is that it saves to a phone even with no internet. The downsides are that it cannot be changed after printing, and it makes a much denser, busier code because there is more data to encode.

My honest take: link a URL you control, like a contact page. It is a lighter, cleaner code, and you can edit the details on the page anytime without touching the card. A vCard locks your details into the ink the moment you hit print.

One note on this tool. It builds URL and chat codes (Website link, WhatsApp, Telegram, iMessage), so it does not currently generate a raw vCard. For a vCard you would use a dedicated vCard encoder. Better still, point a URL code at a contact page that has a "Save contact" button, which gets you the offline-save convenience without freezing your details into the print.

Once you know where it points, building the code takes under a minute. Open the tool, land on the Website link tab, and:

  1. Paste your URL. Your site, your contact page, your booking link, whatever you decided on above.
  2. Add your logo (optional). Drop a small logo in the center. The tool raises the error correction level automatically so a modest logo still scans. If you want the full rundown on how big is safe, I wrote a QR code logo size guide.

Everything happens in your browser. No account, nothing uploaded, and the finished code is static, so there is no server that can go dark on you later.

The right size for a business card

Business cards get scanned from close range, roughly 4 to 6 inches away, so the code does not need to be large. Around 0.6 to 0.8 inch (1.5 to 2 cm) is plenty for a phone held over the card.

The one rule you cannot bend is the quiet margin, the empty light border around the code. Scanners need that clear space to lock onto the pattern. Crop it to squeeze the code into a tight layout and you will get cards that photograph fine and refuse to scan. If you want the reasoning and the math behind scan distance, it lives in my QR print size guide.

Placement and contrast

Put the code where it has room to breathe. The back of the card or a bottom corner works well. Keep it off the edges, because a code that bleeds to the trim line loses the quiet zone it needs.

For contrast, go dark code on light stock. That is what phone cameras read fastest. A dark-on-white or dark-on-cream code reads far better than anything low-contrast or reversed, and the tool warns you in plain English before download if your colors are too close to call.

Before a big run: test-scan an actual proof, not just the file on your screen. Print one card, scan it with a couple of different phones, and confirm it lands where it should. Catching a problem on one proof is free. Catching it on a box of 500 is not.

Make it survive a job change

This is the real argument for a URL over a vCard. Point the code at a page you own, and when your title, number, or email changes, you update the page. The code on the card stays exactly the same, but everyone who scans it sees your current details. Same card design, new information behind it.

A vCard cannot do that. The details are printed into the pattern, so a new job means new cards, full stop.

And because a code made here is static, it never expires. The link is baked into the pattern, not routed through a subscription that can lapse. So the code itself keeps working for years, and the page it points to is yours to edit whenever life changes.

FAQ

What should a business card QR code link to?

A page you control, like your site or a contact page, so you can update it without reprinting. A vCard works too but cannot be edited after printing.

How big should the QR code be on a business card?

About 0.6 to 0.8 inch (1.5 to 2 cm), with the quiet margin left intact. Cards are scanned up close, so it does not need to be large.

URL or vCard, which is better?

A URL for most people, because you can edit the destination page later. A vCard only if you want the contact to save with no internet and never change.

Will the code still work if I change jobs?

If it points at a page you control, yes, just update the page. A vCard code would need new cards.

Can I add my logo?

Yes, the tool centers it and raises error correction so it still scans. Keep it modest in size.

Do business card QR codes expire?

Static ones do not. Made here, the link is baked into the code, so there is nothing that can lapse.
