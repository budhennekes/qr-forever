How to Make a QR Code for Instagram

By Bud Hennekes, who built QR Codes Made Easy. Updated July 2026.

Make a static QR code that opens your Instagram profile (or a specific post, or your link-in-bio) in the app. Copy your Instagram URL, paste it into the tool, and print the code anywhere. Because it is static, it never expires, and unlike Instagram's own in-app QR code, this one matches your brand and prints sharp at any size. Free, no account.

Decide what it should open

A QR code for Instagram is really just a QR code that holds your Instagram URL. So the first question is which URL. You have three good options.

  • Your profile. The address is https://instagram.com/yourusername. This is the right pick for a plain "follow us" code. Scanning it drops people on your profile so they can hit follow.
  • A specific post or reel. Open the post, grab its instagram.com/p/... or instagram.com/reel/... URL, and use that. Good for evergreen content you want to keep pointing people at, like a tutorial or a launch announcement.
  • Your link-in-bio or a landing page you control. This is the smartest choice for flyers and campaigns, because you can change where that page sends people without reprinting a single code. The code stays the same. The page behind it does the moving.

If you are not sure, start with the profile. It is the option most people mean when they say "QR code for my Instagram."

Make the static code (2 minutes)

Once you know which URL you want, the rest takes a couple of minutes.

  1. Copy the Instagram URL you chose from step one.
  2. Open the tool and paste it into the Website link tab. That tab encodes any URL, and an Instagram address is just a URL.
  3. Add your brand colors and a center logo if you want. Dropping a logo in the middle automatically bumps the error correction to the highest level, so the code still scans with your mark on it.
  4. Download a PNG (up to 4000px, print-ready) or a true vector SVG for large print.

No account, nothing uploaded. The whole thing runs in your browser, and the link gets baked into the pattern rather than routed through my server. Before you download, a plain-English contrast warning fires if your colors would be too faint to scan, and the quiet-zone margin around the code is always included so a scanner can lock on.

This vs Instagram's built-in QR code

Instagram already has its own code. You find it under Settings, then QR code (the feature that used to be called Nametag). It works fine for showing a friend across the table.

The catch is that it lives inside the app. It is Instagram-branded, you cannot recolor it, and you cannot export a clean vector for a large print job. A static code you make here is different on all three counts: it matches your brand, it prints sharp at any size, and it can point wherever you want, including a page that is not Instagram at all. If you want one code on a flyer that sends people to a landing page today and your profile next month, the built-in code cannot do that. A code pointed at a link-in-bio page on your own site can.

Where to put it

Once it is printed, the code goes wherever your customers already look.

  • Storefront windows and door decals
  • Product packaging and product tags
  • Business cards (here is the business card walkthrough if that is your use case)
  • Market stalls and event booths
  • Flyers and posters

Whatever you put it on, add three words under it: "Scan to follow." That tiny label does most of the work. People need to know why they would point their camera at a square before they bother.

Why static, and one thing to watch

A static code encodes your Instagram URL directly, so there is no vendor redirect sitting in the middle that can expire the day a subscription lapses. That is the whole reason I only make static codes. If you want the longer version of that argument, I wrote it up in do QR codes expire.

There is one thing to watch. If you point the code at your profile, your username is baked into the pattern. Rename your account later and that printed code needs reprinting, because it still points at the old handle. If there is any chance you will change your username, point the code at a stable link-in-bio page you control instead. Then a rename is just an edit on that page, and every code you printed keeps working.

Test-scan before a big print run. On a phone with Instagram installed, scanning a profile link usually opens the app. Without the app, it opens in the browser instead. That is normal behavior, not a broken code, but scan it on a real phone before you order 500 flyers so you know exactly what your customers will see.

Print size matters too. A permanent link is no help if the square is too small to read across a room. The print size guide has the numbers for scan distance.
