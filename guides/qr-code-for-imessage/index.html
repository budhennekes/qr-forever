How to Make a QR Code for iMessage and Texts

By Bud Hennekes, who built QR Codes Made Easy. Updated July 2026.

Make a static QR code that opens a new text message to your number, with an optional message already typed. On Apple devices it opens as a blue iMessage; on everything else it is a normal SMS. Make it (the iMessage tab)

A text-message QR code encodes an sms: link to your number. When someone scans it, their phone opens the Messages app with your number already filled in, so all they have to do is type (or send, if you pre-loaded the text). Nothing to log into, nothing to install.

Here is the whole thing, start to finish:

  1. Open the tool and click the iMessage tab.
  2. Pick your country so the number is formatted right.
  3. Enter your phone number.
  4. Add an optional message you want pre-typed (skip this if you just want a blank text to open).
  5. Style it if you like, then download the PNG or SVG, or hit Copy code for your website to grab an embed snippet.

That is it. The code you get is static, which means the link is baked into the black and white pattern itself. It is made right in your browser, nothing gets uploaded, and there is no account.

iMessage vs a plain text (the honest version)

Here is the part most QR tools skip. There is no Apple link that forces iMessage specifically. Apple does not publish an "open iMessage" URL, so nobody can encode one, me included.

What actually happens is simpler. The standard sms: link opens the Messages app. From there, Apple decides the color on its own: when an Apple device texts another Apple device, the message turns blue and rides over iMessage. When an Apple device texts an Android phone, it turns green and goes as a normal SMS. The QR code does not choose blue or green. The two phones do.

So the practical version: this one code works for everybody. On an iPhone texting your iPhone, it is a blue iMessage. On an Android phone, or an iPhone texting an Android, it is a regular text. It is just not iMessage-exclusive, because that option does not exist. If your audience is mixed, label the sign "text us" rather than "iMessage us" and you will never confuse anyone.

If you would rather route people into a chat app that does have its own link, I built the same thing for WhatsApp and for Telegram. Those open a specific app by design. Plain texting does not, and that is fine, because everyone already has a texting app.

Add a pre-typed message

You can pre-load the message body so the person just presses send. Type something like Hi, I would like to book in the message field and it rides along inside the code. When they scan, the text is already sitting in the box waiting.

This is the difference between "text us" and "text us, and here is the exact thing to say." It works well for:

  • Bookings. Pre-fill "I'd like to book a table for" and let them finish the sentence.
  • Orders. Pre-fill "Order for pickup:" so you get a clean, sortable inbox.
  • Questions. Pre-fill "Question about" so you know why they are texting before you open it.

Keep the pre-typed text short and let them add the specifics. A long pre-fill feels like a form.

One honest caveat. Whether the pre-typed message shows up depends on the person's phone and messaging app. Most fill it in without a problem. A few may open a blank message to your number instead. The number always fills in, so the worst case is they see an empty text to the right person and type it themselves. Nothing breaks either way.

Where to put it

A text code earns its keep anywhere a phone call would feel like too much and an app download would feel like way too much. People will text who would never call.

  • A sign or window decal. "Text to book" outside a salon or a barber.
  • A menu or table tent. "Text for today's specials."
  • A flyer or yard sign. "Text us with questions" beats printing a number people have to type by hand.
  • Your website. Use the Copy code for your website button to drop a self-contained snippet onto a contact page.

The rule of thumb: the more casual the ask, the better texting does. "Text us with questions" gets answered. "Call for an appointment" gets ignored.

Why it never expires

This code encodes the sms: link directly, the same way a static website code encodes a URL. There is no vendor server sitting in the middle, so there is nothing that can lapse, get switched off, or start charging you a subscription to keep working.

That is the whole reason I only make static codes. A dynamic code routes every scan through a company's redirect, and the day that account stops paying, the printed code goes dead. A static text code has no middleman to die. Print it on a sign, leave it up for three years, and it still opens a text to your number. I get into the full static-versus-dynamic story in do QR codes expire if you want the details.

Before every download, the tool also runs a plain-English contrast check (so a pale code on a pale background does not fail to scan) and always includes the quiet-zone margin around the pattern, which scanners need to lock on. You do not have to think about either one. Make the code, put it where people are, and it works.

FAQ

How do I make a QR code for iMessage?

Open the iMessage tab in the tool, enter your number and an optional message, then download the code. It opens a text to you. Free, no account, and it never expires.

Is there an iMessage-only QR code?

No. Apple does not offer an iMessage-only link, so no tool can make one. The standard text link opens the Messages app, which shows up as a blue iMessage when both phones are Apple devices, and as a green SMS otherwise.

Will it work on Android?

Yes. On Android the code opens the SMS app with your number already filled in, so it works for everyone. It just arrives as a normal text instead of a blue iMessage.

Can I pre-fill the text?

Usually yes. You can pre-type a message so the person just presses send. A few phones may open a blank message to your number instead, but the number always fills in, so the code still works.

Does a text-message QR code expire?

No. It encodes the sms link directly, so there is nothing in the middle that can lapse. Unlike a dynamic code that routes through a vendor's server, a static text code has no subscription to keep alive.

What should I write on the sign?

"Text us" is the safest wording for a mixed audience, since the code arrives as a blue iMessage on Apple devices and a green SMS everywhere else. Skip "iMessage us," which only fits part of your audience.
